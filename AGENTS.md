# Repo Instructions

## Highest Priority

This project is written across multiple AI sessions. Continuity, causality, and clean handoff are more important than novelty.

- Do not write a new chapter by memory alone.
- Do not contradict established facts for the sake of surprise.
- If a detail is uncertain, verify it in the repo before using it.

## Required Reading Before Writing A New Chapter

Read these files before drafting any new chapter:

- `world.md`
- `style.md`
- `characters.md`
- `outline.md`
- `notes/series-state.md`
- `notes/continuity-ledger.md`
- The most recent 1-2 chapter files in `chapters/`

## Required Updates After Writing A Chapter

Any AI that writes or revises a published chapter must update all of the following in the same turn:

- `chapters/chapter-XXX.md`
- `docs/chapters/chapter-XXX.html`
- `docs/index.html`
- Previous/next chapter navigation in adjacent chapter pages
- `notes/series-state.md`
- `notes/continuity-ledger.md`

If one of these is skipped, the handoff is incomplete.

## Continuity Rules

- Treat `notes/continuity-ledger.md` as the hard continuity ledger.
- Treat `notes/series-state.md` as the author-facing summary and next-step guide.
- Record new hard facts separately from guesses or atmosphere.
- Record any new timeline anchor, evidence item, character-state change, and unresolved question introduced by the chapter.
- Do not resolve a long-line mystery unless the existing structure clearly supports that reveal.

## Writing Rules For Handoff Safety

- Keep anomalies fragmentary and falsifiable.
- Preserve the distinction between hard evidence, witness statement, and protagonist perception.
- If a chapter introduces a new clue, also record where it came from and whether it is stable or possibly contaminated.
- If a chapter changes the risk profile of a character, write that down explicitly in the handoff files.
